# Universal Git Visualiser — Intro Video Script
**Format:** Talking head, 60–90s, LinkedIn/X. Target: ~75 seconds spoken.
**Style:** Direct to camera, conversational — not a pitch-deck voice. You're telling a friend what you built and why, not reading ad copy.

---

## HOOK — 0:00–0:10
*(To camera, no screen yet — grab attention before you show the product)*

> "Every time I open a new GitHub repo, I do the same annoying thing: clone it, `cd` into it, dig through five READMEs to find the actual run command, and half the time it's still wrong. So I built a tool that skips all of that."

---

## THE WHY — 0:10–0:22
*(Still to camera — this is the part people actually remember)*

> "I didn't build this because I thought the world needed another dev tool. I built it because I was losing time — real time — every single week just figuring out how to run other people's code before I could even evaluate it. If you've ever cloned a repo just to see what's inside and then deleted it five minutes later, this is for you."

---

## FEATURE 1 — Instant run/deploy commands — 0:22–0:37
*(Cut to screen recording: paste a repo URL, commands appear)*

> "Paste any GitHub link, and it instantly tells you exactly how to run, build, or deploy that specific project — the real command, ready to copy, not a generic guide you have to adapt yourself."

---

## FEATURE 2 — Browse without cloning — 0:37–0:52
*(Cut to screen recording: file tree expanding, clicking into a file, markdown rendering)*

> "You can explore the entire file structure and read the docs right in your browser — no cloning, no cluttering your machine with a repo you're not even sure you want yet. See what's inside before you commit to anything."

---

## FEATURE 3 — Security / trust — 0:52–1:08
*(Back to camera, or split-screen with the privacy banner visible)*

> "And here's the part I actually care most about: nothing runs through a server I control. It's entirely client-side — your browser talks directly to GitHub. So if you're pasting in a token to check a private company repo, that token — and your code — never touches anything but GitHub's own API."

---

## CTA — 1:08–1:15
*(To camera)*

> "It's free, it's open, and it's live right now — link's below. Go paste a repo in and see what you're actually working with before you clone a single thing."

---

## Notes for filming
- Cut hard between "to camera" and "screen recording" segments — don't dissolve, it kills pacing in a scroll-feed video.
- The `demo.gif` sequence (forcedotcom/sf-skills → ingest → SKILL.md → both cards → copy) covers Feature 1 and 2 almost exactly — you can lift those exact screen segments instead of re-recording separately.
- If you're cutting for time, the WHY section is the first thing to trim, not the features — but don't cut it entirely. It's the difference between "another tool" and "a tool built by someone who had your exact problem."
- Say "GitHub" out loud at least twice — feed algorithms and search both key off it, and it's a trust signal for devs skeptical of yet another tool touching their code.
